块级作用域

let，const 和 var 的根本区别不用多说，主要由下：

  • 块级作用域(let const)和函数作用域(var)
  • 全局变量(let const)和顶级对象 window 属性(var)
  • 暂时性死区(let const)和变量提升(var)
  • 不允许在同一作用域内重复声明同一变量(let const)

const

我们知道 const 定义的常量是不能改变的，这个不能改变指的是常量的内存地址，基本类型变量保存在栈内存中，而引用类型是将对象在堆内存中的地址保存在栈中，里面的值保存在堆中。

const 声明只能保证栈区的内存地址不发生变化，如何使对象成为真正的常量呢？

Object.freeze()

Object.freeze()方法可以冻结一个对象。具体来说冻结指的是：

  • 不能向这个对象添加新的属性
  • 不能修改其已有属性的值
  • 不能删除已有属性
  • 以及不能修改该对象已有属性的可枚举性、可配置性、可写性。

成功冻结了一个对象，但如何处理嵌套对象呢？可以使用递归遍历对象并冻结。

Proxy

使用 Proxy 可以定义对象的 set 方法来保证对象不被更改。

这种方法同样无法冻结嵌套对象，需要进行递归冻结。

Object.defineProperty

Object.defineProperty 和 Object.preventExtensions 可以将对象属性变得不可配置以实现对象冻结。

箭头函数

箭头函数相比普通函数写法简洁了不少，但两者的区别也是相当大的。

  • 声明方式
  • this 指向
  • 箭头函数不能用作构造函数，根本原因是没有自己的 this
  • 箭头函数没有自己的 arguments 对象
  • 箭头函数没有 prototype
  • 箭头函数不能用作 Generator 函数，不能使用 yeild 关键字

for…of

在 for…of 之前，ES5 已经有 for，forEach 和 for…in 三种 for 循环了。

相对于 for，for…of 遍历数组或类数组更加方便；相对于 forEach，for…of 可以使用 break 或 continue；相对于 for…in，for…of 不需要 index 就可以访问数组值。

Promise

promise 有三种状态：

  • Pending(进行中)
  • Resolved(已完成，又称 Fulfilled)
  • Rejected(已拒绝)

一个 promise 的状态只可能从“等待”转到“完成”态或者“拒绝”态，不能逆向转换，同时“完成”态和“拒绝”态不能相互转换。

promise 必须实现 then 方法，否则一直会是 pending 或者 rejected(出错情况下) 状态，promise 可以执行多次 then，如果执行过程中出错，便会跳到最后的 catch。

then 方法接受两个参数，一个是成功的回调，一个是失败的回调，当然失败的回调可以换成 catch。

Promise 的 all 方法提供了并行执行异步操作的能力，并且在所有异步操作执行完后才执行回调。

Promise 的 race 方法和 all 相反，promise 队列中只要有一个 promise 完成，就会执行回调，之后队列中的其他 promise 才会完成。

Proxy

Proxy 意思为代理，也就是说在你访问对象之前，需要先经过 Proxy 的处理。

Proxy 实例接受两个参数：target 和 handler。

target 参数表示所要拦截的目标对象，handler 参数也是一个对象，用来定制拦截行为。

常用的 handler 包含 get 和 set 两种行为。

get 用于拦截某个属性的读取操作，接受三个参数：

  • target：目标对象
  • propKey：属性名
  • receiver（可选）：proxy 实例本身（严格地说，是操作行为所针对的对象）

set 用于拦截某个属性的赋值操作，可以接受四个参数。

  • target：目标对象
  • propKey：属性名
  • value：属性值
  • receiver（可选）：Proxy 实例本身

为了更方便地操作对象，ES6 还引入了 Reflect。

  • 只要 Proxy 对象具有的代理方法，Reflect 对象全部具有，以静态方法的形式存在。这些方法能够执行默认行为，无论 Proxy 怎么修改默认行为，总是可以通过 Reflect 对应的方法获取默认行为。
  • 修改某些 Object 方法的返回结果，让其变得更合理。比如，Object.defineProperty(obj, name, desc)在无法定义属性时，会抛出一个错误，而 Reflect.defineProperty(obj, name, desc)则会返回 false。
  • 让 Object 操作都变成函数行为。某些 Object 操作是命令式，比如 name in obj 和 delete obj[name]，而 Reflect.has(obj, name)和 Reflect.deleteProperty(obj, name)让它们变成了函数行为。

Object 和 Reflect 常见不同：

Reflect(target, name, property) 返回布尔值，相较于 Object.defineProperty(target, name, property) 语义更好。

Reflect 使用 has 方法判断对象中是否含有某属性。

当然，Reflect 的主要用途是配合 Proxy 使用，因为他们的参数形式是一样的。

我们可以使用 Proxy 来实现观察者模式：

一个对象可以使用 Object.setPrototypeOf 继承另外一个对象，但不能继承多个对象。

使用这样的方式可以使一个对象继承多个对象。

Reflect

通过 Proxy 我们了解到了 Reflect 的一些用法和作用，现在我们来细细的看看 Reflect。

为什么要设计 Reflect 呢？主要有以下原因：

  • 将 Object 对象的一些明显属于语言层面的方法放到 Reflect 对象上
  • 修改某些 Object 方法的返回结果，使其更加合理
  • 让对象操作都变成函数行为
  • Reflect 对象的方法和 Proxy 对象的方法一一对应，只要是 Proxy 对象的方法，都能在 Reflect 对象上找到对应的方法。这就让 Proxy 对象可以方便地调用对应的 Reflect 方法完成默认行为

Reflect 提供的方法如下：

Reflect.apply ( target, thisArgument, argumentsList )

等同于 Function.prototype.apply.call( target, thisArgument, argumentsList )

不用说，这就等同于Object.defineProperty()，用于定义或修改对象属性，返回一个布尔值表示是否操作成功。其对应的 Object 方法如果操作失败的话，会抛出异常。

deleteProperty

删除对象中的某个属性，等同于delete obj.a

get

Reflect.get ( target, propertyKey [ , receiver ] )

Set，WeakSet

Set 是一种叫做集合的数据结构，主要用于数据重组。

Set 的成员是唯一且无序的，没有重复的值，成员可以是任何类型的唯一值，无论是基本类型还是对象引用。

Set 提供了 4 种操作方法和 4 种遍历方法：

  • add(value)：新增，相当于 array 里的 push。
  • delete(value)：存在即删除集合中 value。
  • has(value)：判断集合中是否存在 value。
  • clear()：清空集合。
  • keys()：返回一个包含集合中所有键的迭代器。
  • values()：返回一个包含集合中所有值得迭代器。
  • entries()：返回一个包含 Set 对象中所有元素得键值对迭代器。
  • forEach(callbackFn, thisArg)：用于对集合成员执行 callbackFn 操作，如果提供了 thisArg 参数，回调中的 this 会是这个参数，没有返回值。

WeakSet 对象允许你将弱引用对象储存在一个集合中。

WeakSet 对象中存储的对象值都是被弱引用的，如果没有其他的变量或属性引用这个对象值，则这个对象值会被当成垃圾回收掉。

相较于 Set，WeakSet 只提供了 4 种操作方法，没有遍历方法：

  • add(value)：新增，相当于 array 里的 push。
  • delete(value)：存在即删除集合中 value。
  • has(value)：判断集合中是否存在 value。
  • clear()：清空集合。

当我们需要一个可以及时垃圾回收的弱引用集合时，可以选择 WeakSet。

Map，WeakMap

Map 是一种字典型数据结构，主要应用于数据存储。

Map 采用键值对结构存储，具有极快的查找速度。

Map 提供了 5 种操作方法和 4 种遍历方法：

  • set(key, value)：向字典中添加新元素。
  • get(key)：通过键查找特定的数值并返回。
  • has(key)：判断字典中是否存在键 key。
  • delete(key)：通过键 key 从字典中移除对应的数据。
  • clear()：将这个字典中的所有元素删除。
  • Keys()：将字典中包含的所有键名以迭代器形式返回。
  • values()：将字典中包含的所有数值以迭代器形式返回。
  • entries()：返回所有成员的迭代器。
  • forEach()：遍历字典的所有成员。

WeakMap 对象是一组键值对的集合，其中的键是弱引用对象，而值可以是任意。

和 WeakSet 不同的是，WeakMap 的键名是弱引用对象，但键值依然是可以正常引用的。

WeakMap 中，每个键对自己所引用对象的引用都是弱引用，在没有其他引用和该键引用同一对象，这个对象将会被垃圾回收（相应的 key 则变成无效的），所以，WeakMap 的 key 是不可枚举的。

和 WeakSet 一样，WeakMap 没有遍历方法，只有 4 个操作方法：

  • has(key)：判断是否有 key 关联对象。
  • get(key)：返回 key 关联对象（没有则则返回 undefined）。
  • set(key)：设置一组 key 关联对象。
  • delete(key)：移除 key 的关联对象。
