常见兼容问题

解决 IE9 以下的浏览器不能使用 opacity：filter:alpha(opacity=50);

不同浏览器的默认 margin 和 padding 不同：*{margin: 0;padding: 0;}

事件：

  • 添加事件：addEventListener/attachEvent
  • 移除事件句柄：removeEventListener/detachEvent/elem['on'+type]=null
  • 取消默认事件：preventDefault/returnValue=false
  • 阻止冒泡：stopPropagation/cancelBubble=true
  • 非 chrome 浏览器 scrollTop：document.documentElement.scrollTop
  • new Date：2019-1-1这样的格式不能在所有浏览器中都用new Date(str)生成正确的日期对象，应该用2019/1/1

BOM

BOM(浏览器对象模型)，它的核心是 window 对象，表示当前浏览器实例

BOM 中有许多对象

  • Document
    • document 既属于 DOM 也属于 BOM 对象，document 由一系列集合构成，可以访问文档的各个部分，指向当前窗口内的文档节点。
  • Location
    • Location 对象包含有关文档当前位置的信息，表示载入窗口的 URL，还可以解析 URI。
    • 应用: 导航到一个新页面。
  • Navigator
    • Navigator 包含大量 Web 浏览器相关的信息。
    • 应用: 检测浏览器名称及版本。
  • Screen
    • 表示一个屏幕窗口，往往指的是当前正在被渲染的 window 对象，可以获取用户屏幕相关的信息。

BOM 和 DOM 有什么区别

DOM 是文档对象模型，用来获取或设置文档中的标签及其属性。

BOM 是浏览器对象模型，用来获取或设置浏览器的属性、行为。例如：新建窗口、获取屏幕分辨率、浏览器版本号等。

HTML

行内元素和块级元素

行内元素

行内元素有：span, a, br, img, i, input, textarea, select 等

  1. 和其它元素排在一行
  2. 高度、宽度以及内边距都是不可控的
  3. 宽高就是内容的高度，不可以改变
  4. 行内元素只能包含行内元素，不能包含块级元素

块级元素

块级元素有：address, h1-h6, hr, p, pre, ul, ol, dl, table, form, div 等

  1. 总是从新的一行开始
  2. 高度、宽度都是可控的
  3. 宽度没有设置时，默认为 100%
  4. 块级元素中可以包含块级元素和行内元素

JavaScript

DOM 事件

DOM 事件分为：DOM0 级事件，DOM2 级事件，DOM3 级事件。

DOM0 级事件是最原始的事件处理方式，将事件定义在标签内，这种方式的事件函数名和 JS 中的函数名有强烈的耦合性，修改不方便。同一事件无法绑定多个函数。

DOM2 级事件弥补了 DOM0 的缺点，可以通过 addEventListener 对同一事件绑定多个函数。

DOM3 级事件没有新增绑定事件的方法，但是添加了许多事件类型。比如页面滚动事件(scroll)，取得，失去焦点事件(focus, blur)，滚轮事件(mousewheel)，键盘事件(keydown,keypress)等。

网络

HTTP 协议

HTTP 协议，也就是超文本传输协议。

HTTP 基于 TCP/IP 通信协议来传递数据(HTML 文件，图片，查询结果)等等。

HTTP 属于应用层的面向对象协议，工作于客户端-服务端架构之上。

HTTP1.0,HTTP1.1,HTTP2.0 协议对比如下：

Range

Range 是针对 HTTP1.0 的浪费现象的一个解决方案。如：断点续传，有了 Range，客户端可以知道本地下载的进度，转而向服务器请求剩下的文件信息。节约了用户的时间，也防止服务器重复地传数多于数据造成的带宽浪费。

长连接

HTTP1.0 的时候，浏览器和服务器只能保持短暂的连接，浏览器对服务器的每次请求都要建立一个 TCP 连接，TCP 连接需要三次握手，如果页面中有太多的资源需要请求就会大大降低访问速度。HTTP1.1 使用长连接解决了这个问题，超时无内容传输的时候才会断开连接，多个请求可以使用一个 TCP 连接，提升了访问速度。

多路复用

多路复用是 HTTP2.0 使用的技术，就是同一个连接并发处理多个请求，而且并发请求的数量比 HTTP1.1 大了好几个数量级。
chrome 浏览器每个页面最多允许建立 6 个 TCP 连接，发起请求->服务端处理->响应请求这样算是完成了一个浏览器请求。
如果一个页面发起太多请求可能会把 6 个 TCP 连接全部占用，阻塞后面的请求，降低响应速度。
HTTP2 采用多路复用指的是在同一个域名下，开启一个 TCP 的连接，请求以 stream 的方式传输，每个 stream 都有唯一标识，一旦建立了连接，后续的请求都可以复用这个连接，并且可以同时发送，服务器端根据 stream 的唯一标识来对应相应的请求，解决了请求阻塞的问题。

数据压缩

HTTP2.0 支持 HPACK 算法对头部数据进行压缩，数据体积小了传输速度会更快。

服务器推送

如果我们对支持 HTTP2.0 的 web server 请求数据的时候，服务器会顺便把一些客户需要的资源一起推送到客户端，免得客户再次创建连接请求服务器，适合加载静态资源。

cache

Cache-Control 表示浏览器使用缓存，不向服务器发请求。

只有请求相同的 url 才会利用之前的缓存，那么我在 http 的请求路径上加上一个查询参数就可以了，每次更新 css，js，只要变化查询参数就行了。

Expires

HTTP1.0 中出现了 Expires，值为格林威治时间，表示资源的过期时间，但这个时间依赖于本地时间，如果本地时间修改了，可能造成缓存失败。

Cache-Control

Cache-Control 出现于 HTTP / 1.1，优先级高于 Expires ,表示的是相对时间。

ETag

ETag 是 HTTP1.1 协议请求头中的一个字段。

当浏览器请求服务器的某项资源，服务器根据资源用 MD5 算出一个哈希值，并通过 ETag 返回给浏览器，浏览器吧 ETag 的值和资源同时缓存在本地，如果下次再次请求相同的资源，会通过 If-None-Match 字段将 ETag 发给服务器，服务器再次计算资源的哈希值，如果相同就返回一个 304 表示未修改，这时命中协商缓存，浏览器调用本地缓存。通过 Cache-Control 可以节省服务器的带宽。

为什么使用 ETag：

  • 一些文件也许会周期性的更改，但是他的内容并不改变(仅仅改变的修改时间)，这个时候我们并不希望客户端认为这个文件被修改了，而重新 GET
  • 某些文件修改非常频繁，比如在秒以下的时间内进行修改，(比方说 1s 内修改了 N 次)，If-Modified-Since 能检查到的粒度是 s 级的，这种修改无法判断(或者说 UNIX 记录 MTIME 只能精确到秒)
  • 某些服务器不能精确的得到文件的最后修改时间

Last-Modefied

Last-Modefied 代表服务器端文件的最后修改时间，当浏览器发起请求时，会向服务端发送 If-modefied-Since 报头，询问在 Last-Modefied 之后有没有被修改过。如果没有修改过，则返回 304 使用缓存，如果修改过，则向服务器请求资源，返回 200。

但是如果本地打开缓存文件，会导致 Last-Modefied 被修改，所以在 HTTP/1.1 中引入 Etag。

强缓存和协商缓存

浏览器在加载资源的时候会根据请求头的 Expires 和 Cache-Control 判断是否命中强缓存，如果命中，直接从缓存读取资源，不向服务器发起请求。

如果强缓存没有命中，浏览器一定会像服务器发起请求，通过 Last-Modefied 和 ETag 判断资源是否命中协商缓存，如果资源没有更新，返回 304，浏览器读取缓存中的资源，如果没有命中，返回 200，从服务器加载资源。

如何选择合适的缓存策略

  1. Cache-Control —— 请求服务器之前
  2. Expires —— 请求服务器之前
  3. If-None-Match (Etag) —— 请求服务器
  4. If-Modified-Since (Last-Modified) —— 请求服务器

TCP 和 UDP

为什么要四次挥手？

  • 第一次挥手，客户端向服务器发送 FIN 报文请求断开连接
  • 第二次挥手，服务器向客户端发送 ACK 报文表示收到请求
  • 第三次挥手，服务器向客户端发送 FIN 报文表示可以断开连接
  • 第四次挥手，客户端向服务器发送 ACK 报文表示收到

为什么第二次和第三次不能合并成一次挥手呢？

因为服务器收到 FIN 报文之后，可能还需要发一些信息给客户端，不能马上断开连接，因此需要服务器操作完毕再发送 FIN 报文给客户端。

然而，如果服务器端也没有信息需要发送，ACK 和 FIN 会合并成一次挥手。

如果只有三次挥手，就不能支持半关闭了，当客户端执行 close 函数后会向服务器端发出 FIN 消息，服务器端操作系统接收到该信息后向服务器进程发出 EOF(end of file)，这时服务器无论调用 send 还是 recv 都会失败，此时服务器端认为客户端已经关闭了连接，因此调用 close 函数关闭同客户端的连接，当客户端接收到 FIN 信息后进行 ACK 回复确认，整个四次挥手的过程完毕。

为什么需要三次握手？

  • 第一次握手，客户端发送 SYN 请求报文，并进入 SYN SENT 状态，等待服务器确认。
  • 第二次握手，服务器接收到 SYN 报文后，返回 ACK 并且发送一个 SYN 报文，并进入 SYN RCVD 状态。
  • 第三次握手，客户端收到 ACK 和 SYN 后，需要返回一个应答 ACK 报文，此时该连接进入半连接状态的队列，当服务器端收到 ACK 后，双反进入 ESTABLISHED 状态。

第一次握手发送报文丢失怎么办？

客户端发送报文之后会启动一个定时器，超时之后未收到服务器的确认，就会再次发起 SYN 请求，尝试时间是上一次的两倍，如果总尝试时间为 75 秒，则建立连接失败。

为什么第二次握手要法 ACK 和 SYN 两个报文？

因为 TCP 是全双工通信，协议规定收到建立连接请求后必须返回一个序列号，同时建立本端到对端的通信连接，也叫做捎带应答机制。

第二次发送报文丢失怎么办？

在发送完 ACK+SYN 报文后会启动一个定时器，超时没有收到 ACK 确认，会再次发送，会进行多次重试。

AJAX

XMLHttpRequest 的缺陷

XMLHttpRequest 标准分为两个版本

老版本主要缺陷如下：

  • 受同源策略的限制，不能发送跨域请求
  • 不能发送二进制文件（如图片、视频、音频等），只能发送纯文本数据
  • 在发送和获取数据的过程中，无法实时获取进度信息，只能判断是否完成

新版本对老版本的缺陷进行了优化，新增了以下功能：

  • 可以发送跨域请求，在服务端允许的情况下
  • 支持发送和接收二进制数据
  • 新增 formData 对象，支持发送表单数据
  • 发送和获取数据时，可以获取进度信息
  • 可以设置请求的超时时间

单工，半双工，全双工

单工：只支持数据在一个方向上传输；在同一时间只有一方能接受或发送信息，不能实现双向通信，举例：电视，广播。

半双工：允许数据在两个方向上传输,但是,在某一时刻,只允许数据在一个方向上传输,它实际上是一种切换方向的单工通信；在同一时间只可以有一方接受或发送信息，可以实现双向通信。举例：对讲机。

全双工：允许数据同时在两个方向上传输,因此,全双工通信是两个单工通信方式的结合,它要求发送设备和接收设备都有独立的接收和发送能力；在同一时间可以同时接受和发送信息，实现双向通信，举例：电话通信。

同步和异步

AJAX

AJAX 是一个创建交互式网页应用的开发技术，可以在不用刷新浏览器的情况下，与服务器进行异步通讯技术。

AJAX 的核心是 JavaScript 对象 XmlHttpRequest。

XmlHttpRequest 的作用是向服务器提出请求并处理响应，而不阻塞用户操作，可以在页面加载以后进行页面的局部更新。

XmlHttpRequest 常用方法和属性：

  • open(method, url, boolean, username, password)
    • 该方法创建 http 请求
    • 第一个参数是指定提交方式(post、get)
    • 第二个参数是指定要提交的地址是哪
    • 第三个参数是指定是异步还是同步(true 表示异步，false 表示同步)
    • 第四和第五参数在 http 认证的时候会用到。是可选的
  • send(content)
    • 如果是 get 方式，并不需要填写参数，或填写 null
    • 如果是 post 方式，把要提交的参数写上去
  • setRequestHeader(header, value)
    • 设置消息头（使用 post 方式才会使用到，get 方法并不需要调用该方法）
  • getAllResponseHeaders()
  • getResponseHeader(header)
  • abort()

AJAX 实现流程：

  • 创建 XMLHttpRequest 对象,也就是创建一个异步调用对象
  • 创建一个新的 HTTP 请求,并指定该 HTTP 请求的方法、URL 及验证信息
  • 设置响应 HTTP 请求状态变化的函数
  • 发送 HTTP 请求
  • 获取异步调用返回的数据
  • 使用 JavaScript 和 DOM 实现局部刷新

HTML

meta 标签都有什么作用

  • 页面关键词
    • <meta name="keywords" content="关键词" />
  • 页面描述
    • <meta name="description" content="想要描述的内容" />
  • 搜索引擎索引方式
    • <meta name="robots" content="index,follow" />
    • all：文件将被检索，且页面上的链接可以被查询
    • none：文件将不被检索，且页面上的链接不可以被查询
    • index：文件将被检索
    • follow：页面上的链接可以被查询
    • noindex：文件将不被检索
    • nofollow：页面上的链接不可以被查询
  • 页面重定向和刷新
    • <meta http-equiv="refresh" content="0;url=" />
  • 申明编码
    • <meta charset='utf-8' />

除此之外还有很多。

哪些标签的使用可以有利于 seo

<h> 标签对搜索引擎非常友好，它告诉了搜索引擎这个页面的主题是什么，但是权重一般只到<h3>为止，<h3>以下的标签将不会对 seo 产生影响。
