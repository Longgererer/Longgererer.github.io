注意：再执行 eject 之前不可以更改项目文件，否则会报错

执行完后目录中多了 config 和 script 文件夹

现在开始分析工程目录：

config

config 目录下放的是项目配置文件

env.js

env.js 文件获取客户端的环境，一般有生产环境和开发环境两种

所以 env.js 的主要目的在于读取 env 配置文件并将 env 的配置信息给到全局变量 process.env

modules.js

modules.js 文件毫无疑问就是用来获取项目所需 module 的

path.js

path.js 主要作用是来格式化路径，还有配置提供给项目的各种路径，包括构建路径，public 路径等等

pnpTs.js

pnpTs.js 文件的作用就是将一些安装了的 modules 暴露，以便 import 引入

webpack.config.js

这个一看就知道是 webpack 的配置文件

下面说一下这个文件的常用配置：

getStyleLoaders

getStyleLoaders 方法中定义了一些样式文件方法，如 style-loader,css-loader,postcss-loader，用于导入 css

module

module 中可以配置样式私有化

在 getStyleLoaders 传入对象中配置 modules: true 就可以实现 css 样式私有化

public

在来看下面的文件：

public 文件夹主要是用于存放一些静态文件，比如图片，还有不需要进行打包的 css，js 文件等

script

script 文件夹主要是包含了构建项目所需的命令，还有单元测试文件

src

src 是最重要的部分，也是我们项目的主要操作文件夹，一般 src 目录结构类似于下面这样：

api

api 文件夹用来放置项目所需的一些公共的 api

common

common 文件夹顾名思义就是一个放公共样式和方法的文件夹

components

components 用来放置项目组件

pages

pages 文件夹用来放置项目的一个个页面

router

router 文件夹用来放置路由，通常文件夹里面有一个 index.js

store

store 用来进行状态管理，如果项目不需要 redux，那么可以忽略这个文件夹
