那么，console.log是如何得到a值的呢? 这里就涉及到了JS的查询机制

在JS中有两种查询方式：LHS 和 RHS，他们其实是Left Hand Side和Right Hand Side的缩写，意为左手边和右手边
什么意思呢?就是一个赋值操作的左边和右边，下面我详细的介绍一下它俩的区别

LHS查询

LHS查询的目的在于寻找变量的容器本身

我们还是用a来打比方

这里使用了LHS查询，找到a容器，并将2 “装” 进去，也就是赋值给它

疑问来了：这行代码中a不是刚声明就赋值了么，为什么还要LHS去寻找a容器呢?

正如问题所说，a刚声明就赋值了，这里就有一个非常值得注意的地方：声明和赋值是两个操作

JS编译器会将 var a = 2 分成两部分处理：

var a
a = 2

编译器在遇到 var a 的时候首先会查询当前作用域是否已经存在一个名为a的变量，若是，则忽略该声明，若不是，则当前作用域会创建变量a
然后遇到 a = 2 如果当前作用域存在变量a，进行赋值操作，若不存在变量a，则沿着作用域链进行查找

如果最终找到了a，便赋值给它，如果没有，反手就抛出一个异常

实际上，就算没有前面的声明，JS引擎也不会抛出异常，因为在非严格模式下，注意，是非严格模式下，LHS查询如果查到顶层作用域还没有找到变量a，会自动在顶层作用域创建变量a来交给引擎处理

小结

这里有一道题，如果你看得出那些地方用了LHS，那些地方用了RHS说明你已经掌握了它们

想看答案么?

三处用了LHS查询，四处用了RHS查询：

LHS查询

  • var c = foo(2) (c的赋值)
  • var b = a (b的赋值)
  • a = 2 (这里有一个隐式赋值，将2传给函数，函数传参是按值传递，会将2赋给a)

RHS查询

  • foo(2) (foo的调用)
  • var b = a (a的调用)
  • console.log(a + b) (a的调用)
  • console.log(a + b) (b的调用)
