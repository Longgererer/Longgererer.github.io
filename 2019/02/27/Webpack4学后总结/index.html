找时间学了Webpack4，感受颇多，总结一下

安装Webpack4

注意: 在Webpack4之后，Webpack和Webpack-cli就分离了，网站很多教程都是Webpack3的，会导致很多人在执行命令的时候弹出没有安装Webpack-cli包

在全局安装 Webpack 和 Webpack-cli ， g 是 global 的意思，代表全局
另外， install 可以简写成 i ，效果一样的

使用Webpack前必须要准备两个文件 package.json 和 webpack.config.js

package.json

package.json这个文件用来管理你在本地安装的npm包
现在项目目录打开命令行，在命令行输入 npm init 创建package.json
之后会弹出一堆问题：

  • package name => 项目名称
  • version => 项目版本号
  • description => 项目描述
  • entry point => 入口文件
  • test command => 当你运行npm run test时运行的命令
  • git repository => 你的git仓库地址
  • keywords => 是一个字符串数组，放便于查找
  • author => 作者
  • license => 许可证，国内一般用MIT许可

这里我只填了 package name 为 project

现在我们尝试安装一个npm包：

  • –save表示将包名和版本储存在package.json的dependencies字段中
  • 如果是–save-dev表示将包名和版本储存在package.json的devDependencies字段中
  • dependencies是运行时所依赖的包，devDependencies是开发时所依赖的包
  • 如果直接npm install ejs，就是在dependencies字段下

装完之后你会发现文件夹中多了一个node_modules文件夹，里面是你所安装的包，将来这个文件夹删了也没关系，可以使用npm install命令来重新安装项目的所有依赖

webpack.config.js

Webpack在执行的时候，除了在命令行传入参数，还可以通过指定的配置文件来执行。默认情况下，会搜索当前目录的webpack.config.js文件，这个文件是一个 node.js 模块，返回一个 json 格式的配置信息对象

在文件夹手动创建一个webpack.config.js文件
在里面输入以下代码：

  • devtool => 表示项目所用调试工具，有很多，我就不说了，但这仅1仅是方便调试，最后打包生成的js文件是非常大的，所以该字段可以删除
  • mode => 用来告知webpack使用相应模式的内置优化，有两个选项：production 和 development , 就是生产模式和开发模式,production环境要求更小的代码量以加快网页的加载速度,development主要是为了开启热加载，路由重定向等功能
  • entry => 表示入口文件路径，可以指定多个文件，同样会打包成多个文件
  • output => 表示出口文件，也就是打包之后的文件信息
  • path => 表示打包文件的输出路径（绝对路径）
  • filename => 表示打包文件的名称

注意事项

  1. 任何json文件，包括package.json,代码中都不能含有注释，可以在文件前面和后面加注释，但不能在中间
  2. package.json中的scripts字段存放的是脚本例如"scripts":{"build":"webpack --mode development"}，那么可以在命令行中输入npm run build相当于执行webpack --mode development命令，更加方便
  3. package.json中的”name”字段不要和任何包名相同，否则安装该包的时候会报错
