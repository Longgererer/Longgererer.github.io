Vuex是一个专为Vue.js应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

Vuex 的基本思想

项目中常常会使多个组件共享一个状态，这样会破坏单项数据流的简洁性

所以Vue将组件的共享状态抽取出来，以一个全局的单例模式进行管理，通过定义和隔离状态管理中的各种概念并强制遵守一定的规则，可以将代码变得更加结构化且易维护

用途

如果不打算开发大型单页应用，使用Vuex 可能是繁琐冗余的。因为Vuex会带来很多概念和框架，如果项目本身就简单，那么使用Vuex反而会使项目变得复杂

安装

npm install vuex --save or yarn add vuex
and
import Vuex from 'vuex'
then
Vue.use(Vuex)

在一个模块化打包系统中，必须使用这样的方法引入Vuex

Getter

有些时候我们需要将 state 中的一些属性过滤掉

如果多个组件都要用到这个函数，那就都要导入这个函数，这种方式不是很理想

Getter就像计算属性一样，返回的值会根据它的依赖存储起来，只有它依赖的发生了改变才会被重新计算

如上，getter接受state作为其第一个参数

Getter会暴露为store.getters对象，可以通过它访问到这些值

Getter也可以将其他getter作为第二个参数

注意，getter在通过属性访问时是作为Vue的响应式系统的一部分缓存其中的。

通过方法访问

可以让getter返回一个函数，来实现给getter传递参数，这在你对store中的数组进行查询的时候非常有用

注意，getter在通过方法访问时，每次都会去进行调用，而不会缓存结果。

Mutation

想要更改Vuex中的store，唯一的方法就是通过mutation,每个mutation都有一个字符串的事件类型和一个回调函数。我们需要在回调函数中更改状态，它会接受state作为第一个参数

这些回调函数不能够直接调用，应该当increment被触发之后，再调用这个函数，使用store.commit来触发函数

提交负荷

可以通过 store.commit 向事件传入其他参数，即 mutation 的载荷

推荐以对象作为载荷，这样可以包含多个字段

提交mutation的另外一种方式是直接使用type属性的对象

Mutation 需要遵守 Vue 的响应规则

Vuex中的store是响应式的，状态发生了变更，那么相关的Vue组件也会更新，这意味着Mutation也需要遵守一些注意事项：

  • 最好提前在你的store中初始化好所有属性
  • 当需要在对象上添加新属性的时候，应该使用Vue.set(obj,'newProp',123) 或者以新对象替换老对象，可以利用对象展开运算符

使用常量替代 Mutation 类型

使用常量替代Mutation是Flux实现的主要方式，这样可以使linter之类的工具发挥作用，把这些常量放在一个单独的文件中，可以让别人对整个app包含的mutation一目了然

Mutation 必须是同步函数

为什么必须是同步函数呢？因为devtools需要捕捉前一状态和后一状态的快照，但是异步回调不会立即触发，devtools不知道什么时候回调才会被调用，在任何回调函数中进行状态改变都是不可追踪的

在组件中提交 Mutation

可以在组件中使用this.$store.commit('XXX')提交mutation，或者使用mapMutations辅助函数将组件methods映射为store.commit调用(需要在根节点注入store)

Action

Action类似于mutation，不同于

  • Action 提交的是 mutation，而不是改变状态
  • Action 可以包含任意异步操作

Action函数接受一个与store实例具有相同方法和属性的context对象，因此你可以调用context.commit来获取state和getters

Module

因为使用单一状态树，当状态过多的时候，store 会变得特别臃肿，Vuex 允许我们将 store 分割成模块，每个模块拥有自己的 state,mutation,action,getter,甚至是嵌套子模块——从上至下进行同样方式的分割

模块的局部状态

对于模块内部的 mutation 和 getter，接受的第一个参数是模块的局部对象

对于模块内部的action，局部状态通过context.state暴露出来，根节点状态则为context.rootState

对于模块内部的getter，根节点状态会作为第三个参数暴露出来

命名空间

默认情况下，模块内部的 action、mutation 和 getter 是注册在全局命名空间的——这样使得多个模块能够对同一 mutation 或 action 作出响应。

如果希望你的模块具有更高的封装度和复用性，你可以通过添加 namespaced: true 的方式使其成为带命名空间的模块。当模块被注册后，它的所有 getter、action 及 mutation 都会自动根据模块注册的路径调整命名。
