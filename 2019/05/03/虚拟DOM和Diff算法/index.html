React有两个核心概念：虚拟DOM和Diff算法

首先我们来看看什么是DOM：DOM意思是文档对象模型（Dcoument Object Model），它是一个结构化文本的抽象。对于Web开发者，这个文本是一段HTML代码，DOM也就被叫做HTML DOM。HTML的元素在DOM中变成了节点。DOM是浏览器中的概念，用JS对象表示页面上的元素，并提供了操作DOM对象的API

那么，什么是虚拟DOM呢？

虚拟DOM

首先说一下，这个虚拟DOM不是React发明的，但是React用了它且免费提供。它是轻量的，是从浏览器特定（Browser-specific，这里意指特定的浏览器需要特定的实现）实现细节中提取出来的。由于DOM本身就已经是一个抽象了，所以虚拟DOM，实际上，是一个抽象的抽象。它允许React跳过既慢又限于特定浏览器的真实DOM操作，以在这个抽象世界中做自己的计算。

虚拟DOM:(Virtual Document Object Model)

在React框架中，采用JS对象来模拟页面上的DOM和DOM嵌套，虚拟DOM可以让页面上的元素高效更新：

  1. 虚拟DOM不会进行排版与重绘操作
  2. 虚拟DOM进行频繁修改，然后一次性比较并修改真实DOM中需要改的部分（注意！），最后并在真实DOM中进行排版与重绘，减少过多DOM节点排版与重绘损耗
  3. 真实DOM频繁排版与重绘的效率是相当低的
  4. 虚拟DOM有效降低大面积（真实DOM节点）的重绘与回流，因为最终与真实DOM比较差异，可以只渲染局部（同2）

真实的DOM远远没有这个看上去那么简单，打个比方，虚拟DOM就像是内存一样介于CPU和硬盘之间，由于硬盘速度实在是太慢，CPU不能直接改变硬盘中的数据，需要将修改后的数据保存在内存中，最后在将所有改动写入到硬盘里

真实的DOM如果发生了变动需要重新绘制整个的DOM树，这绝对是性能杀手，为了解决这个问题，虚拟DOM通过Diff算法比较出新旧两个DOM树的不同，从而只修改被改变的元素，而不会使得整个DOM树重绘

Diff算法

Diff算法主要经历以下几个阶段

逐级对比

逐级比较（tree diff）：React会用逐级的比较新旧两个DOM树之间的差异，当整颗DOM逐层对比完毕，所有需要更新的元素，必然能够找到

组件对比

在进行tree diff的时候，每一层中组件级别的对比叫做组件对比（component diff）

  • 如果对比前后组件的类型相同，则暂时认为此组件不需要更新
  • 如果对比前后组件的类型不相同，则需要移除旧组件，创建新组件，并追加到页面上

元素对比

在进行组件对比的时候，如果两个组件类型相同，则进行元素级别的对比
