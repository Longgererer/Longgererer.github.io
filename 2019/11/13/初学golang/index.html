定义数量级

整形

在 go 中常用 uint 和 int 两种整形，两种整形都有相应的 8，16，32 以及 64 位，不同在于 uint 最小值为 0，而 int 的范围包含负数，比如：

  • uint8:无符号 8 位整型（0~255）
  • int8：有符号 8 位整型（-128~127）

特殊整形

  • uint: 32 位系统上就是 uint32，64 位上就是 uint64
  • int: 和 uint 同理
  • uintptr: 无符号整型，用于存放一个指针

注意：32 位的机器上只能最高使用 32 位的整型，无法使用 64 位

go 语言为了区分 ASCII 和其他类型的字符，将英文的字符叫 byte，中文及其他字符叫 rune

rune 只是一个名称而已，内部存储 rune 采用的是 int32 存储的，那么实际上占用 32 个字节

要修改字符串，首先要将其转换成 []rune 或者 []byte，然后再转换为 string，无论哪种转换最后都要重新分配内存，并复制字节数组

切片

切片是一个拥有相同类型元素的可变长度的序列，它是基于数组类型的一层封装，它非常灵活，支持自动扩容。切片是一个引用类型，它的内部结构包含地址、长度、容量，切片一般用于快捷操作一块数据集合

切片数组都是基于底层数组，是引用类型的。底层数组的元素值改变了，其他切片数组的值都会改变

make 函数可以构造切片，制定切片的长度和容量

上面的语句使用 make 定义了一个切片，第一个参数是类型，第二个是长度，第三个是容量，如果第三个参数省略则默认和第二个参数一样

切片的本质就是一个框，圈住了一块连续的内存。属于引用类型，真正的数据都是保存在底层数组里的

所以不能将两个元素相同的切片进行比较，因为这两个切片内存地址不相同

但是切片可以与 nil 进行比较，一个 nil 值的切片没有底层数组，切片的长度和容量都是 0，但是一个容量和长度都是 0 的切片不一定是 nil

所以一个切片是否为空要根据长度来判断而不是等不等于 nil

由于切片拷贝后两个切片共享同一个底层数组，一个切片的改变会影响另一个切片

至于切片的遍历，既可以使用 for 也可以使用 for range，和遍历数组一样

GO 函数支持多个返回值

GO 函数内是不能够再定义函数的

在 defer 归属的函数即将返回时，将延迟处理的语句按 defer 定义的逆序进行执行，也就是说，先被 defer 的语句最后被执行，最后的 defer 语句最先执行

注意：GO 语言中函数的 return 不是原子操作，在底层是分为两步来执行：第一步是返回值赋值，第二步是返回真正的 RET。函数中如果存在 defer，那么 defer 执行的时机是在第一步和第二步之间

返回值是 6，因为 f1 函数首先将 5 赋给 x，再执行 defer，x++之后值为 6

返回值是 5，因为 f1 函数首先将 x 赋给返回值，再执行 defer x++，但是返回值是 5，而不是之后的 x

因为传入 defer 执行函数的 x 是一个副本，而不是外部的 x

函数也可以作为返回值

所以，如果在操作数据库的过程中出了问题，应该先将数据库连接断开，再报错
