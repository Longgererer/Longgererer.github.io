JS 的内存管理对大部分前端人员来说可能很陌生，因为 JS 这门语言会在变量不再使用的时候自动释放内存，这让很多开发者认为完全不用关心内存管理，这是错误的想法

JS 内存空间分为栈(stack)、堆(heap)、池(一般也会归类为栈中)。 其中栈存放变量，堆存放复杂对象，池存放常量

我们来看一个例子

在栈内存中的数据发生复制行为时，系统会自动为新的变量分配一个新值。var b = a 执行之后，a 与 b 虽然值都等于 20，但是他们其实已经是相互独立互不影响的值了。

引用类型的复制同样也会为新的变量自动分配一个新的值保存在栈内存中，但不同的是，这个新的值，仅仅只是引用类型的一个地址指针。当地址指针相同时，尽管他们相互独立，但是在堆内存中访问到的具体对象实际上是同一个

了解了这些，答案应该一目了然了

内存的生命周期大致是这样的：

  • 分配所需内存
  • 使用分配的内存
  • 不需要的时候释放内存

虽然 JS 可以自动释放不再使用的变量，但前提是开发者要让它知道该变量已经作废了，这就需要确认哪些内存是可以回收的哪些不可以

垃圾回收

我们来看看如何垃圾回收

但也有一种特例，就是循环引用

这种情况下两个对象之间相互引用，就算函数执行完毕也不会垃圾回收,甚至可能会导致内存泄漏

那么 V8 引擎使用什么算法进行回收呢？

Scavange 算法：通过复制的方式进行内存空间管理，主要用于新生代的内存空间

Mark-Sweep 算法和 Mark-Compact 算法：通过标记来对堆内存进行整理和回收，主要用于老生代对象的检查和回收。

现代浏览器的算法大致是这样的：

将不再使用的对象定义为无法达到的对象，凡是能从根部到达的对象，都是需要使用的，反之就是要垃圾回收的，当 div 和事件处理函数不能从全局对象触及的时候，就会被垃圾回收

内存泄露

对于持续运行的服务进程（daemon），必须及时释放不再用到的内存。否则，内存占用越来越高，轻则影响系统性能，重则导致进程崩溃。不再用到的内存，没有及时释放，就叫做内存泄漏

所以想要避免内存泄漏，仍然需要开发者确认不再需要的内存并主动释放
