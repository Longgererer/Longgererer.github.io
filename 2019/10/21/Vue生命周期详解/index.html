我们可以通过 Vue2 的源码来理解 beforeCreate 的机制：

在 beforeCreate 之前执行了几个初始化方法：initLifecycle，initEvents 以及 initRender

initLifecycle 在实例上初始化了 $children 和 $refs 以及其他属性

initEvents 在实例上初始化了 _events 和 _hasHookEvent

initRender 在实例上初始化了 $vnode，$slots 和 $scopedSlots，其作用主要是创建虚拟节点

在这三个方法执行完后才会执行 beforeCreate 函数，这个时候 data、watch、computed 这些属性都还没有初始化，当然不能被操作

created

created 代表实例已经创建完成，但是还没有挂载，所以除了 $el 之外，其他都是可以获取到的

上述方法在 beforeCreate 和 created 之间执行

initInjections 用于初始化 inject 选项

initState 初始化了 data、props、methods、computed 和 watch 选项

initProvide 初始化了 provide 选项

页面上显示 a 的值已经成功更新为 2，$el 仍然是未定义状态

准备挂载

在 created 之后，就需要准备挂载的工作了，在这之前需要先进行判断，实例是否含有 el 选项，如果没有指定该选项就不需要进行挂载执行，如果后续要进行挂载，需要通过 $mount 方法挂载

假设含有 el 选项或者调用了 $mount 方法，还需要判断是否含有 template 选项，如果含有该选项，需要将 template 编译成 render 函数，render 函数是用来将模板和 data 数据编译成 html。如果没有 template 选项，就将外部的 HTML 作为模板编译，也就是在 template 标签中写的 HTML，最后会在内存中生成一个编译好的模板

beforeMount

beforeMount 在挂载之前被调用，相关的 render 函数首次被调用，此时模板已经编译好了，但是还存放在内存中，没有正式挂载到页面上

在 beforeMount 之后会将内存中编译好的模板，也就是 $el 挂载到 el

mounted

在 $el 挂载完毕后会调用 mounted 方法，页面上生成了真正的 DOM，这里我们就可以进行 DOM 进行操作了

注意：mounted 只能保证当前的组件已经挂载完毕，而不能保证其所有子组件也都挂载完毕

同样地，如果想要在这里对子组件进行一些操作，可以使用 this.$nextTick：

在 Mounted 的过程中，如果 data 对象的数据发生了改变，就会执行这个生命周期函数，一般在这里访问 DOM 比较合适

这样是不会触发 beforeUpdate 的，因为 vue 的 observer 检测到新值和旧值是一样的，是不会做任何操作的，也就不会触发了

其次，由于没有在 data 上初始化的数据不会被动态观测，所以同样不会触发：

第三，在 mounted 之前的生命周期函数中改变 data 数据也是不会触发 beforeUpdate 的，换言之，只有真正改变了渲染到页面上的数据才会触发该生命周期函数

updated

在 updated 中，vue 会根据虚拟 DOM 原则，找到最便捷的更新页面 DOM 结构的最小路径，将改变更新到 DOM 上面，完成更新，和 mounted 一样，updated 只能保证当前组件更新完毕，而不能够保证子组件更新完毕，可以使用 this.$nextTick 来替换掉 updated

只有执行了 beforeUpdate 方法才会执行 updated 方法，也就是说触发 updated 之前必须满足触发 beforeUpdate 的条件

beforeDestroy

beforeDestroy 在实例被销毁之前调用，在这个函数中依然可以获取实例中的属性

当用户离开页面或跳转页面的时候会触发，如果使用了 keep-alive 对组件进行了缓存，就不会触发

destroyed

destroyed 用于销毁 Vue 实例，同时实例上绑定的事件，数据都会被移除，包括子组件，可以在这个函数内清除先前定义的计时器
