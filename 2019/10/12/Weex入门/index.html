Weex 通用样式

在了解 Weex 内容之前先来看看 Weex 的样式规范：

尺寸单位

Weex 对于长度值目前只支持 px 和 wx（不受屏幕宽度和 viewPortWidth 影响），不支持相对单位（em、rem）以及 %、pt

如果不希望受屏幕宽度和 viewPortWidth 影响的尺寸，请使用 wx 单位

所有元素宽高尺寸默认为 0

颜色

Weex 支持的颜色单位如下：

  • 十六进制颜色
  • RGB 颜色
  • RGBA 颜色
  • 字面量颜色，如 red

不支持 HEX 和其他格式的颜色

盒模型

Weex 的盒模型基于标准 CSS 盒模型，每个元素都可以视作为一个盒子

Weex 内置组件

a

和 H5 一样，页面跳转使用 a 标签，不同的是，a 标签内不能直接写文本，需要使用 text 标签添加文本

注意：href 属性值必须是一个 Weex 页面的 URL，否则会是一个未定义行为

div

div 使用时的注意事项：

  • 不要在 <div> 中直接添加文本，而要使用 <text> 组件
  • 在 Weex 中，<div> 不可滚动
  • 不要产生过多的嵌套，建议不要超过 14 层，否则会很影响页面性能

text

  • text 组件不支持其他子组件，只能包含文字
  • 如果想在 text 组件中添加空格，必须通过数据绑定的方式，直接添加会被过滤

image

  • <img> 标签因为存在兼容性原因，所以将在新版本中删除，建议使用 <image> 标签
  • Weex 没有内置的图片缓存库，可以使用 SDWebImage 和 Picasso，在使用 <image> 之前，需要在 native 侧先接入相应的 adapter 或者 handler
  • <image> 不支持内嵌子组件
  • <image> 必须指定样式中的宽高

可在 <image> 上绑定 load 事件：

list

<list> 组件是提供列表功能的组件，在 <list> 内使用循环生成的 <cell> 填充：

<list> 中只能包含以下几种组件：

  • <cell>
  • <header>
  • <refresh>
  • <loading>

除了上面这几个，只有通过 fix 定位的组件才能正确渲染

cell

<cell> 必须以一级组件的形式存在于 list、recycler、waterfall 中

<cell> 中可以添加任意类型的子组件，但是不能再添加滚动容器了

loading

<loading> 元素必须在被 <scroller>、<list>、<waterfall> 包含的时候才能被正确渲染

<loading> 可以渲染 <text> 和 <image> 之类的组件

组件 loading-indicator 作为 <refresh> 和 <loading> 子组件使用，它拥有默认的动画效果实现。

可以使用 loading 事件使 <scroller>、<list>、<waterfall> 被上拉完成时触发加载事件

refresh

<refresh> 为容器提供下拉刷新功能

<refresh> 是 <scroller>、<list>、<waterfall> 的子组件，只能在被它们包含时才能被正确渲染

display 属性控制 <refresh> 组件的显示和隐藏，可选值为 show/hide

可以在 <scroller>、<list>、<waterfall> 被下拉完成时触发 refresh 事件，或者触发 pullingdown 事件，通过 event 参数获取以下数据：

  • dy: 前后两次滑动距离值
  • pullingDistance: 下拉的距离
  • viewHeight: refresh 组件高度
  • type: “pullingdown” 常数字符串

recycle-list

<recycle-list> 是一个列表容器，具有回收和复用的能力，可以大幅优化内存占用和渲染性能

<recycle-list> 只能使用 <cell-slot> 作为其直接子节点，使用其他的节点是无效的

cell-slot

<cell-slot>是每一项对应的模板，并不对应实际节点，只用来描述模板结构

for

在 <recycle-list> 不需要使用 v-for 循环列表，而是使用 for 属性，但是 for 循环的只是自己内部的子节点，而不是当前节点，这是不同之处

switch

在 <recycle-list> 属性用于指定区分子模版类型的字段名，语义和编程语言的 switch 一致：

如果想将自定义组件作为 <recycle-list> 的子组件，那么需要给 <template> 标签定义一个 recyclable 属性，才可以用在 <recycle-list> 中：

这样定义的自定义组件仍然可以用在其他地方

注意：

<recycle-list> 中不允许使用函数调用，也不允许使用 filter 过滤器，推荐使用 computed 来实现 filter 的效果，因为每次取值都做一次通信的话会大幅降低性能

slot 不允许在 <cell-slot> 中使用，因为功能有部分重叠

<recycle-list> 中的组件都不是虚拟 DOM！与虚拟 DOM 相关的功能也不支持，尽量不要操作节点

scroller

<scroller> 是一个容纳子组件进行横向或者竖向滚动的容器组件

  • <scroller> 不允许相同方向的 <list> 或者 <scroller> 互相嵌套，也就是说，在嵌套的情况下，必须一个是横向一个是竖向，方向不能相同
  • <scroller> 需要显示地设置他的宽高，可以使用绝对定位或者直接设定宽高值

想要在滚动到底部的时候加载更多信息可以绑定 loadmore 方法

silder

<slider> 主要被用于轮播图中

<slider> 支持任意子组件，也可以包含一个 <indictor> 组件用于显示轮播指示器

indictor

indicator 只能在 slider 中使用

textarea

<textarea> 和 <input> 相似，可用于接收用户输入数据，支持多行输入

input

<input> 有多种类型：

  • text
  • password
  • url
  • email
  • tel

waterfall

<waterfall> 组件是提供瀑布流布局的核心组件，是一种比较流行的页面布局

<waterfall> 组件只能包含以下几种组件或是 fix 定位的组件：

  • <cell>
  • <header>
  • <refresh>
  • <loading>

video

Weex 中内置的 <video> 组件一般只做 demo 用途，在你的 App 中，你应该用你喜欢的视频库重新实现 video 组件

web

<web> 可以使 H5 与 Native 元素结合，用于在 WEEX 页面中显示由 src 属性指定的网页内容

richtext

<richtext> 是富文本组件，可以嵌套 <span> <a> <image>，也只有这三个组件可以包含在富文本组件中
