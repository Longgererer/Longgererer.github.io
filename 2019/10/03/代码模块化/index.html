使用 AMD 和 CommonJS 模块化

AMD

AMD 是构建客户端 Web 应用程序的 Javascript 流行工具之一，AMD 可以很容易地指定模块及依赖关系

AMD 提供名为 define 的函数，它接收以下参数：

  • 新创建模块的 ID，使用该 ID，可以在系统的其他部分引用该模块
  • 当前模块依赖的模块 ID 列表
  • 初始化模块的工厂函数，该工厂函数接受依赖的模块列表作为参数

首先我们使用 define 函数定义 ID 为 MouseCounterModule 的模块，因为该模块依赖于 jQuery，因此 AMD 要先请求 jQuery 模块，这个请求是异步执行的，以避免阻塞，所有的依赖下载并解析完成之后，调用模块的工厂函数，并传入所有依赖的模块，在本例中只有一个 jQuery 依赖，所以只传入一个 jQuery 依赖

在工厂函数内部，是与模式类似的创建模块的过程，创建暴露模块公共接口的对象

AMD 优点如下：

  • 自动处理依赖，无需考虑模块引入顺序
  • 异步加载模块，避免阻塞
  • 在同一个文件中可以定义多个模块

CommonJS

AMD 和 CommonJS 的设计理念不同，前者设计明确基于浏览器，而后者则是面向通用 Javascript 环境，CommonJS 在 Node.js 中饱受欢迎，它是基于文件的模块，所以一个文件只能定义一个模块

CommonJS 提供变量 module，该变量具有属性 exports，通过 exports 可以很容易的扩展额外属性，最后，module.exports 作为模块的公共接口

因为文件同步加载，可以访问模块公共接口，这是 CommonJS 在服务器端更流行的原因，模块加载相对更快，只需要读取文件系统，而客户端需要从远程服务器下载文件，同步加载就意味着阻塞

CommonJS 有两个优点：

  • 语法简单，只需要定义 module.exports 属性，引入模块也只需要 requier 函数
  • CommonJS 是 Node.js 默认的模块格式，所以我们可以使用 npm 上成千上万的包

ES6 模块

ES6 模块结合了 CommonJS 和 AMD 的优点：

  • 与 CommomJS 模块类似，ES6 模块语法相对简单，并且基于文件
  • 与 AMD 类似，ES6 模块支持异步模块加载

ES6 引入了两个关键字：export 和 import
